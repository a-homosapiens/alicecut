# 使用手册 — AliceCut

把 `.lrc` 歌词和歌曲音频变成抖音/视频号风格的「动态歌词视频」：逐字卡点、错落排版、整句翻转/上移转场，导出带音轨的竖屏或横屏 MP4。

本手册面向使用者，覆盖 GUI 的全部功能与命令行自动化用法。架构与实现原理见 [DESIGN.md](DESIGN.md)。

## 1. 安装与启动

环境要求：Node.js ≥ 18（开发用 24 验证），Windows / macOS（Linux 可运行，CI 见 §13）。

```bash
npm install            # 安装依赖（含 ffmpeg，无需单独装）
npm run fonts          # 可选：下载内置开源字体（中文、英文、正风毛笔）
npm run dev            # 启动桌面应用
```

不执行 `npm run fonts` 也能用——字体列表会回退到系统自带中文字体（微软雅黑、黑体、楷体等）；内置字体缺失时会静默跳过。

其他命令：

| 命令 | 作用 |
|---|---|
| `npm test` | 运行核心逻辑单元测试 |
| `npm run typecheck` | TypeScript 类型检查 |
| `npm run build` | 生产构建（输出到 `out/`） |
| `node scripts/smoke-export.js` | ffmpeg 编码管线冒烟测试 |
| `npm run benchmark:export` | 8 秒静态/视频工程的软件与 GPU 端到端导出对比（先执行 `npm run build`） |

## 2. 五分钟上手

1. 点顶栏**「导入歌词」**，选一个 `.lrc` 文件（`samples/demo.lrc` 可以拿来练手）
2. 点**「导入音频」**，选对应的 mp3（可跳过——没有音频则输出无声视频）；想要视频背景就再点**「导入视频」**
3. 按**空格**播放，看默认的「逐字弹出」效果
4. 在右侧样式面板换一种特效、换字体、调字号
5. 点顶栏**「导出视频」**，选保存位置，等进度条走完

## 4. 导入素材

### 4.1 歌词（.lrc）

支持的格式特性：

| 特性 | 示例 | 行为 |
|---|---|---|
| 标准行时间戳 | `[00:12.34]歌词` | 行内逐字时间按字符自动插值（标点几乎不占时长） |
| 毫秒级时间戳 | `[00:12.345]` | 支持 |
| 一行多时间戳 | `[00:12.00][01:30.00]副歌` | 展开为两行，按时间排序 |
| 增强型逐字标签 | `[00:12.00]<00:12.00>你<00:12.40>好` | 直接使用精确逐字时间（QQ 音乐等导出的格式） |
| 元数据 | `[ti:歌名][ar:歌手][offset:500]` | 歌名/歌手用于片头；offset 正值=歌词整体提前 |
| GBK 编码 | — | 自动识别回退，无需手动转码 |

空内容的时间戳行（如 `[01:00.00]`）被当作「清屏行」，用于间奏处让上一句退场。

行的结束时间默认 = 下一行的开始时间（最后一行按字数估算，上限 8 秒），之后可在时间轴上任意微调。

### 4.2 音频

支持 mp3 / wav / m4a / aac / flac / ogg，可一次选多个。每个音频成为时间轴**音频轨上的一条线段**（§6.4）：可左右拖动起点、设置重复次数或无限循环；多条音轨导出时自动混音。预览可听歌对卡点，导出混入 AAC 192k 音轨。

### 4.3 背景视频

「导入视频」支持 mp4 / mov / m4v / webm / mkv / avi / flv / wmv / ts / mpg / 3gp，可一次选多段。每段视频成为时间轴**视频轨上的一条线段**，画面按 cover 方式铺满整个画布（保持比例居中裁切），歌词文字画在视频之上。视频线段同样可拖动起点、循环 n 次或无限循环；多段视频按时间排布，重叠时后开始的盖在上面。视频自带的声音**不会**进入成片（要配乐请单独导入音频）。

> **导入归一化**：预览/导出靠 Chromium 的 `<video>` 解码，它只认 MP4/WebM 容器与 H.264/VP8/VP9/AV1 编码。导入不支持的视频（mkv/avi 容器，或 ProRes/HEVC 等编码）时，会用内置 ffmpeg 自动转成 H.264 MP4——能直接复制视频流的（如 mkv 里是 H.264）走快速重封装，否则转码（ProRes/HEVC 等，较慢）。转换结果缓存在用户数据目录，同一文件再次导入直接命中。

### 4.4 字体

「样式面板 → 文字 → 导入字体文件…」支持 ttf / otf / woff / woff2。导入的字体立即可选，但**不会**保存进工程文件——换机器打开工程需重新导入。

### 4.5 图片与资源库

「样式面板 → 背景 → 图片 → 选择图片…」选的图片会自动登记进**图片库**（可导入任意多张，不止一张），随工程保存/打开。

顶栏「资源库」打开一个浮动面板，一览当前工程里的全部资源：

| 分区 | 内容 | 点击一行 | 「移除」 |
|---|---|---|---|
| 视频 | 时间轴上的全部视频线段 | 选中该线段并把播放头跳到其起点 | 删除该线段（不删源文件） |
| 音频 | 时间轴上的全部音轨线段 | 同上 | 同上 |
| 字幕组 | 全部字幕组（含主字幕组），同 §3 的字幕组列表 | 展开该字幕组的面板 | 删除整个字幕组（主字幕组不可删除） |
| 图片 | 图片库里全部已导入图片，缩略图 + 文件名，当前背景图高亮 | 设为当前背景图 | 从图片库移除；若正是当前背景图，背景一并清空（「导入图片…」按钮补充导入） |

资源库主要是**浏览与管理**，不是新的导入入口——视频/音频仍从顶栏「导入视频」/「导入音频」导入（这样才会按现有规则接在时间轴/分层上）；图片库例外，因为资源库本身就是它目前唯一的管理界面。

## 5. 预览与播放

- **空格**：播放 / 暂停（焦点在输入框时不触发）
- 播放条拖动：跳转任意时间
- 预览画布的内部分辨率始终等于导出分辨率，所见即所得；窗口缩放只影响显示大小
- 没有导入音频时也能播放（按歌词时间轴静默走带）

## 6. 时间轴编辑

播放条下方是时间轴：**每句歌词是一个线段**，横向位置和长度对应它的开始/结束时间，颜色按特效区分，线段上显示歌词文本和特效名。

### 6.1 选择

| 操作 | 效果 |
|---|---|
| 点击线段 | 单选，并把播放头跳进该句（预览立即可见） |
| Ctrl + 点击 | 多选 / 取消单个 |
| 字幕组「选中全部字幕」或 Ctrl+A | 选中所有字幕组的全部字幕（不包含独立文字块） |
| Esc / 点击时间轴空白处 | 取消选择 |
| 左侧歌词列表点击某行 | 同步选中对应线段 |

选中状态影响三件事：右侧特效面板作用于选中项（§7.3）、预览画布可拖动文字（§9）、时间轴工具栏出现时间微调输入框。

### 6.2 调整时间

| 操作 | 效果 |
|---|---|
| 拖动线段中部 | 整体挪动开始+结束时间；多选时整组一起挪，保持相对间距；最早一行不会被挪到 0 之前 |
| 拖动线段左/右边缘 | 微调开始/结束时间；**逐字时间按比例重映射**到新区间，节奏不变形；最短 0.1 秒 |
| 单选时工具栏的「开始/结束」输入框 | 精确到 0.01 秒 |
| 时间轴空白处点击/拖动 | 移动播放头（试听） |
| 右上 + / − | 缩放时间刻度；播放时刻度自动跟随播放头滚动 |

线段允许重叠——重叠时两句会同时出现在画面上，是否合理由你判断。

### 6.4 媒体轨（背景视频 / 音轨）

导入视频/音频后，时间刻度下方出现独立的媒体轨：🎬 视频轨在上、🎵 音频轨在下，歌词轨在最下。

| 操作 | 效果 |
|---|---|
| 拖动媒体线段 | 左右挪动它在时间轴上的起点（不能挪到 0 之前） |
| **竖向拖动视频线段** | 换层：视频可以放在多个层上，高层画面盖在低层之上（画中画） |
| 点击媒体线段 | 选中，工具栏出现该线段的编辑控件 |
| 工具栏「✂ 切割」 | 在红色播放头位置把选中线段切成两段（循环线段先按圈展开再切，切出的段保留源区间） |
| 工具栏「开始」输入框 | 精确设置起点（秒） |
| 工具栏「循环 n 次」输入框 | 素材首尾相接重复 n 遍，线段长度 = 修剪区间 ÷ 速度 × n |
| 工具栏「∞」勾选 | 一直循环到项目结束（线段自动延伸到结尾，长度不再计入项目时长） |
| 工具栏「速度」滑杆 | 0.25–4 倍均匀变速；音轨导出时变速不变调（atempo） |
| 工具栏「缩放」滑杆（视频） | 画面缩放（铺满画布为 1.0）；选中后**在预览画布上拖动可平移画面** |
| 工具栏「提取音频」（视频） | 把视频的音频抽取成一条独立音轨线段（与原视频同起点/区间/速度；无音频流会提示） |
| 工具栏「删除」/ Delete 键 | 移除该线段（不删源文件） |

线段上的竖向刻线标出每一遍循环的边界；右侧角标显示 `×n` / `∞` 与变速倍率。

**项目时长** = max(最后一句歌词结束 + 2 秒, 最晚结束的有限媒体线段)。无限循环线段不决定时长，只是填满到结尾。

### 6.5 字幕增删与独立文字块

| 操作 | 效果 |
|---|---|
| 工具栏「+ 字幕」 | 在播放头处插入一句 2 秒的字幕（参与歌词流），双击左侧列表修改文字 |
| 工具栏「+ 文字」 | 在播放头处插入一块 3 秒的**独立文字**：📝 黄色线段单独一行轨道，不参与歌词流（不影响当前行与停靠转场），适合标题/水印/注释 |
| 选中后「删除选中」/ Delete 键 | 删除选中的字幕行或文字块 |

文字块与字幕一样：可拖动起止、可选全部进场/退场特效（停靠式特效演绎为整块进场）、可在画布上拖动位置。

## 7. 特效详解

### 7.1 两类特效

**逐字/逐词进场类**：句内每个字（或词）按自己的时间点依次做进场动画，上一句淡出退场。卡点感来自字的出现节奏。

**整句停靠转场类**：整句作为一个块进场；上一句**不消失**，而是「停靠」到画面上的固定位置（上方堆叠或侧边竖排），再下一句来时继续让位/淡出。画面会同时保留多句歌词。

### 7.2 特效清单

| 名称 | id | 类型 | 单元 | 效果 | 「强度」滑杆控制 |
|---|---|---|---|---|---|
| 无 | `none` | 进场/退场 | 句 | 不做动画；进场立即显示，退场保持到 segment 结束后直接消失 | — |
| 逐字弹出 | `pop` | 进场 | 字 | 缩放过冲回弹 + 上浮淡入 | 过冲与上浮幅度 |
| 缩放冲击 | `punch` | 进场 | 词 | 错落构图（词随机大小/角度），逐词超大砸入回弹 | 砸入的初始放大倍数 |
| 滑动错落 | `slide` | 进场 | 字 | 相邻字从相反方向滑入，带运动模糊 | 滑入距离与模糊量 |
| 打字机 | `typewriter` | 进场 | 字 | 逐字瞬现 + 行尾闪烁光标 | —（无明显作用） |
| 发光渐显 | `glow` | 进场 | 字 | 柔和淡入 + 辉光呼吸脉冲（辉光颜色可在样式面板设置） | 辉光强度 |
| 卡拉OK高亮 | `karaoke` | 逐词高亮 | 词 | 整句一开始全部出现，正在朗读的词放大并染成高亮色（仿 AE/CapCut/Submagic 自动字幕）；高亮色在特效面板「卡拉OK高亮色」设置 | 放大与上浮幅度 |
| 翻转切换 | `flip` | 停靠转场 | 句 | 新句旋转入中心；旧句翻转 90° 竖排停靠侧边（左右交替），与新句垂直居中对齐；保留 1 句 | 停靠旧句的大小 |
| 翻转·底对齐 | `flip-bottom` | 停靠转场 | 句 | 同上，但旧句竖排后的下边缘与新句下边缘对齐（立在同一条底线上） | 停靠旧句的大小 |
| 上移切换 | `rise` | 停靠转场 | 句 | 新句从下方升入中心；旧句缩小停靠在正上方，逐条堆叠，越往上越淡；保留 3 句 | 停靠旧句的大小（拉到 2 时旧句比新句还大） |

新增的 15 个进场效果只显示在「进场」选择器中：

| 名称 | id | 单元 | 效果 |
|---|---|---|---|
| 轻浮上场 | `float-up` | 词 | 从下方轻柔上浮并淡入 |
| 瀑布落入 | `cascade-down` | 字 | 字符依次从上方落入 |
| 变焦显影 | `zoom-focus` | 句 | 由模糊放大状态聚焦成清晰文字 |
| 翻滚入场 | `tumble-in` | 字 | 字符带旋转与回弹滚入 |
| 分侧汇入 | `split-in` | 字 | 左右字符从两侧交替汇合 |
| 疾甩入场 | `whip-in` | 词 | 带倾斜和运动模糊快速甩入 |
| 星散聚合 | `scatter-in` | 字 | 字符从确定性的随机散点聚合 |
| 折叠展开 | `fold-in` | 词 | 词组从压扁折叠状态展开 |
| 霓虹点亮 | `neon-on` | 句 | 模糊闪烁后像霓虹灯一样点亮 |
| 波浪入场 | `wave-in` | 字 | 字符沿波浪轨迹依次归位 |

新增的 15 个退场效果只显示在「退场」选择器中：

| 名称 | id | 单元 | 效果 |
|---|---|---|---|
| 上浮淡出 | `fade-up-out` | 句 | 整句上浮并淡出 |
| 坠落离场 | `drop-out` | 词 | 词组依次向下坠落 |
| 变焦远去 | `zoom-away-out` | 句 | 缩小、失焦并远去 |

| 星散离场 | `scatter-out` | 字 | 字符散向确定性的随机方向 |
| 左甩离场 | `whip-left-out` | 词 | 带倾斜和模糊快速甩向左侧 |
| 翻滚离场 | `tumble-out` | 字 | 字符旋转并向下滚出 |
| 随机溶解 | `dissolve-out` | 字 | 字符按随机顺序逐个溶解 |
| 蒸发消散 | `evaporate-out` | 字 | 字符向上漂移、模糊并消散 |
| 逐词下沉 | `sink-out` | 词 | 词组依次向下沉出画面 |

### 7.3 全局默认 vs 行级特效

- **没有选中任何线段**时，点特效芯片设置的是**全局默认**——所有未单独指定特效的行都用它
- **选中线段后**，特效面板标题变为「应用到选中的 N 条」，点芯片只改选中行；「恢复跟随全局默认」按钮可清除行级设置
- 「无」是明确关闭动画；「恢复跟随全局默认」则是取消行级覆盖、重新使用全局默认特效，两者含义不同
- 多选且各行特效不一致时，所有芯片都不高亮
- 不同特效可以自由混搭（如主歌用上移切换、副歌用缩放冲击）

## 8. 画面样式

| 项 | 说明 |
|---|---|
| 尺寸 | 竖屏 9:16（1080×1920）/ 横屏 16:9（1920×1080）/ 方形 1:1（1080×1080） |
| 背景 | 可折叠的「背景」区块（默认收起，标题右侧显示当前背景摘要）。三选一：**图片**（选本地图片，默认按 cover 铺满画面，可用「图片缩放/水平/垂直」滑杆调整大小与位置，「重置图片位置」回到铺满）/ 渐变（双色线性，可调角度）/ 纯色 |
| 字体 | 点击字体框展开**可视化选择**：首装直接提供得意黑、思源黑体 SC、Inter；其余字体通过轻量 PNG 显示真实字形，并在卡片右下角显示 ⇩。点击后 AliceCut 从 `a-homosapiens/alicecut/font-assets` 下载、注册并缓存，重启后自动恢复，不会在启动时下载或加载全部大字体。内置 34 款字体：原有字体 + samples 中 5 款字体（含朝華標題B、朝華打字機）+ ToronoGlitch 黑体/明朝 H1–H4、正风毛笔、崇羲篆体、思源 SC 与英文字体 + 常见系统中文字体 + 导入文件。可变字重字体会响应粗体按钮和导出样式中的 `fontWeight`。 |
| 字号 / 粗体 / 斜体 | 8–4000px（对数滑杆兼顾常用字号与超大标题）；粗体、斜体开关 |
| 文字整体变换 | 所有文字（歌词与文字块）一起平移（水平/垂直）与旋转，绕画面中心；可一键重置 |
| 文字颜色 / 不透明度 | 文字可半透明（10–100%） |
| 字幕底色 | 文字后面垫圆角色块，颜色任选、不透明度 0–100%（0 = 无底色） |
| 光晕 | 常驻彩色光晕，强度 0–40px，颜色与「发光渐显」特效共用辉光颜色 |
| 阴影 | 投影：不透明度（0 = 关）、颜色、偏移 0–20px、模糊 0–30px |
| 片头显示歌名 | 开启后，第一句歌词出现前淡入显示 `[ti:]` 歌名与 `[ar:]` 歌手 |

> **全局 vs 单句文字属性**：和特效一样，「文字」区块的字体/字号/粗体/斜体/颜色/不透明度——**没选中线段时改的是全局默认**（所有未单独设置的句子一起变）；**选中一句或多句后，区块标题变为「（选中 N 条）」，改动只作用于选中句**，可点「恢复跟随全局默认」清除单句设置。停靠式转场（翻转/上移）暂不支持单句字体覆盖。

## 9. 画布内位置编辑

选中线段且**暂停**时：

1. 预览画面里选中的句子周围出现**虚线编辑框**（仅当播放头处于该句时间内——点线段会自动跳过去）
2. 在画布上按住拖动 → 平移该句的画面位置；多选时一起平移；靠近画面中心时出现**中心参考线并吸附居中**
3. 偏移量随工程保存、导出时生效；虚线框只是编辑指示，不会出现在导出视频里

播放中不可拖动（避免误触）。要恢复居中，把它拖回去即可（偏移以像素记录）。

## 10. 歌词文本编辑

字幕组面板里**双击**某行进入编辑，Enter 确认、Esc 取消。改文本后逐字时间在原时间区间内重新插值（增强型 LRC 的精确逐字时间会丢失，变为均分）。时间不变，只改字。

### 10.1 多字幕组（双语字幕）

用于两种语言（或原文 + 翻译/拼音等）同时显示、竖直错开不重叠的场景：

1. 左侧栏「+ 新增字幕组」新建一个空字幕组，默认以浮动小窗打开
2. 浮动窗里点「导入歌词到此组…」，选一份该语言的 `.lrc`/`.srt`/`.vtt`——不影响已有字幕组
3. 拖动浮动窗标题栏可移动位置；标题栏 ⧉ 收回停靠到左侧栏、✕ 关闭（只是收起面板，字幕组本身还在）
4. 窗内「纵向位置」滑杆调整这组文字的竖直偏移（像素），默认已依次向下错开，一般不用改；
   「显示」勾选框可临时隐藏整组（预览/导出都不显示，不影响播放头位置）
5. 双击顶部名称可改字幕组显示名；「删除字幕组」整组连同它的全部歌词行一起删除（可撤销）

字幕组的字体/字号/颜色等文字样式没有单独的整组设置项：点浮动窗里的「选中此组全部行」，
再到右侧样式面板改（面板已支持"改动只作用于选中行"，见 §8 末尾说明），
即可一次性设置整组文字样式，效果与逐句设置相同。

主字幕组（id 0，最初导入歌词的那组）位置固定居中，不能删除、也没有「纵向位置」/「显示」控件——
和只有一份歌词的旧版工程完全一样。停靠式转场（翻转/上移）历史句数较多时，
额外字幕组的默认间距可能不够，遇到叠字请调大「纵向位置」。

## 11. 导出视频

顶栏「导出视频」→ 选帧率（30 推荐 / 60）、编码格式、容器、速度/画质档位、是否启用硬件加速 → 选保存位置 → 等待。

- **编码格式**：
  - **H.264**（默认，`libx264`）：兼容性最好，几乎所有播放器/平台都吃
  - **H.265 / HEVC**（`libx265`）：同等画质下文件更小，部分老设备/平台播放支持较弱
  - **ProRes**（`prores_ks`）：面向 Mac 剪辑软件（Final Cut Pro / Premiere / DaVinci 等）的中间格式，体积远大于前两者，**只装在 .mov 容器里**——选它后容器会自动锁定为 MOV
- **容器**：MP4（默认）或 MOV；选 ProRes 时容器控件自动禁用并显示 MOV
- **速度 / 画质**：快速 / 均衡（默认，与不调整任何设置时的效果完全一致）/ 高画质三档，
  分别对应更快的编码预设 + 更高的 CRF（画质换速度）或反过来
- **硬件加速**（默认勾选）：H.264 优先走 WebCodecs GPU 驻留路径，Canvas 画面直接编码为
  压缩帧再交给 FFmpeg 混音/封装，避免全尺寸 RGBA 回读与 IPC；其它格式按当前平台探测
  FFmpeg 硬件编码器（Windows: NVENC/QuickSync/AMF；macOS: VideoToolbox）；
  **探测不到会自动、静默回退成软件编码，不会导出失败**，只是速度上没有加成
  （回退信息会打印在应用的终端/控制台里，不在导出弹窗内显示）
- **背景视频精确逐帧取景**（勾选框，只在工程含背景视频时出现，默认不勾选）：默认（不勾选）
  让背景视频正向连续播放来取景，比逐帧精确 seek 快一个数量级；代价是同一份工程重新导出
  两遍时，视频画面可能有肉眼几乎不可见的、亚帧级别的细微差异（文字、音频、无背景视频的
  导出完全不受影响，一直是逐帧确定性的）。如果你需要两次导出的视频画面帧帧字节级一致，
  勾选此项切回精确逐帧 seek（慢很多，见下）
- 输出音频规格：AAC 192k（有音频时）+ faststart，与编码格式选择无关
- 时长：max(最后一句歌词结束 + 2 秒, 最晚结束的有限媒体线段)；无限循环线段在结尾处截断
- 背景视频取景后合入画面，循环/偏移与预览一致；多条音轨按各自起点和循环次数混音
- 软件 fallback 且工程不含背景视频时，纯色、渐变或背景图片只渲染一次；导出帧只包含透明
  文字动画，FFmpeg 循环该静态背景并完成最终合成。WebCodecs 路径则直接在 GPU Canvas 合成
- 导出中可随时取消；预览与导出走同一渲染代码，画面逐像素一致（背景视频在默认「快速」
  取景模式下例外，见上）
- 导出速度取决于分辨率、机器和上面几项设置；1080×1920@30fps 默认设置下大约每秒视频需
  0.5–2 秒渲染。含背景视频的导出曾是最大的速度瓶颈（逐帧精确 seek 的开销远高于纯编码），
  默认的正向连续播放取景已经把这部分成本降低了一个数量级，让含背景视频的导出速度接近
  不含背景视频的水平；勾选「精确逐帧取景」会退回旧的慢速路径。静态背景合成省掉了 Canvas
  每帧重画背景的成本，但当前仍需读取并传输全尺寸 RGBA 文字层，不能完全消除 `getImageData` 瓶颈
- 静态画面通常很容易被 H.264/HEVC 的帧间压缩复用，因此「背景不动」会自然节省码率；透明层
  分离不会直接保证文件更小，最终体积仍主要由编码格式、CRF/画质档、分辨率、帧率以及文字运动量决定
- `npm run benchmark:export` 本机两次实测：8 秒 1080×1920@30 balanced 的 H.264 硬件路径
  对静态背景快 55.1–61.3%，对背景视频快 40.9–48.9%；静态项目与软件成片的逐帧 SSIM
  为 0.996893。不同 GPU/驱动会有差异
- 硬件编码按码率档位控制，软件 x264 按 CRF 控制。几乎全静态的画面用软件编码通常文件更小；
  更看重体积而非速度时可关闭硬件加速。实拍/背景视频在本次 balanced 基准中两者体积接近

## 12. 工程文件

「保存工程」生成 `.alicecut.json`，包含：全部歌词行（含每行的时间、特效、位置偏移、所属字幕组）、
样式设置、额外字幕组列表（§10.1，名称/来源文件/纵向位置/可见性）、图片库（§4.5，全部已导入
图片的路径）、全部媒体线段（视频/音频的**路径**、起点、循环设置——不内嵌媒体数据）。

工程文件同时保存媒体/图片的**绝对路径与相对路径**。相对路径以 `.alicecut.json` 所在目录为基准；
把工程文件连同旁边的 `media/`、`images/` 等素材目录一起复制到另一台电脑后，AliceCut 会优先使用新位置
的相对路径，找不到时才回退到原绝对路径。工程文件不内嵌素材，因此仍需把素材文件一起复制。

「打开工程」恢复一切；两种路径都找不到时会列出缺失路径并跳过。旧版本工程文件可以正常打开
（缺的字段自动补默认值，v1 的 `audioPath` 自动转成一条音轨线段）；v1–v5 工程再次保存后会自动升级为
带相对路径的 v6 格式。

## 13. 命令行 / Pipeline 自动化

GUI 之外，同一个程序支持无头导出，适合批量生产或接入自动化流程：

```bash
npx electron . --export job.json        # 开发环境
alicecut.exe --export job.json          # 打包后
npx electron . --save-project job.json  # 只生成 GUI 可打开的 .alicecut.json 工程
npx electron . --export job.json --save-project job.json  # 同时导出视频并保存工程
```

`--save-project` 后面必须跟 job 路径；即使和 `--export` 一起使用，也要显式写出该值。生成的 `.alicecut.json` 会写到 job 文件同目录，文件名取自歌词文件名。

### job.json 字段

```json
{

  "audio": [
    { "path": "song.mp3", "start": 0, "loop": 1 },
    { "path": "rain-ambience.wav", "start": 5, "loop": "infinite" }
  ],
  "video": [
    { "path": "bg-a.mp4", "start": 0, "loop": 2 },
    { "path": "bg-b.mp4", "start": 16, "loop": "infinite" }
  ],
  "out": "output/video.mp4",
  "fps": 30,
  "duration": 95,
  "gpu": true,
  "hwAccel": "auto",
  "style": {
    "aspect": "9:16",
    "effectId": "pop",
    "fontFamily": "Smiley Sans",
    "fontSize": 92,
    "textColor": "#ffffff", "textAlpha": 0.9, "italic": true, "fontWeight": 700,
    "letterSpacing": 4, "wordSpacing": 12, "lineSpacing": 1,
    "textAlign": "center", "textOrientation": "horizontal",
    "strokeColor": "#000000", "strokeWidth": 0, "strokeAlpha": 1,
    "textBgColor": "#000000", "textBgAlpha": 0.45,
    "halo": 18, "glowColor": "#7dd3fc",
    "shadowColor": "#000000", "shadowAlpha": 0.8, "shadowOffset": 6, "shadowBlur": 10,
    "bgType": "image", "bgImage": "bg.jpg",
    "globalDx": 0, "globalDy": -300, "globalRotate": 12,
    "intensity": 1,
    "showMeta": true
  },
  "lineEffects": { "0-7": "float-up", "8": "punch" },
  "lineEffectsOut": { "0-7": "evaporate-out", "8": "drop-out" },
  "lineEffectDurations": { "0-7": { "in": 0.6, "out": 0.4 } },
  "lineStyles": {
    "0-3": { "textAlign": "left", "letterSpacing": 8, "strokeWidth": 5 },
    "8": { "textOrientation": "vertical", "lineSpacing": 1.2 }
  }
}
```

| 字段 | 必填 | 说明 |
|---|---|---|
| `lrc` | ✔ | 歌词路径；相对路径相对 job 文件所在目录 |
| `out` | 导出视频时必填 | 输出视频路径；目录不存在会自动创建。只使用 `--save-project` 时可省略。`codec:"prores"` 时必须以 `.mov` 结尾，否则直接报错（这是唯一一处硬校验——GUI 里选 ProRes 会自动改容器，但 CLI 的 `out` 是调用方明确指定的路径，写错本该让它失败，而不是被静默改名） |
| `audio` | | 音轨：单个路径字符串、单个对象或数组；省略 = 无声视频。多条音轨自动混音 |
| `video` | | 背景视频：写法同 `audio`；画面 cover 铺满画布，歌词画在其上，视频自带声音不进成片 |
| `duration` | | 成片时长（秒）；缺省 = max(歌词结尾 + 2 秒, 最晚结束的有限媒体线段) |
| `fps` | | 10–60，默认 30 |
| `container` | | `"mp4"`（默认）或 `"mov"`。选 `"prores"` 编码时容器会被强制为 mov（`out` 扩展名仍需自己写对，见上） |
| `codec` | | `"h264"`（默认）/ `"hevc"` / `"prores"`，含义见 §11 |
| `speed` | | `"fast"` / `"balanced"`（默认，省略字段时的行为）/ `"quality"` |
| `hwAccel` | | `"auto"`（探测硬件编码器，找不到自动回退软件）/ `"software"`（默认，直接软件编码不探测） |
| `gpu` | | `true` 保持无头 Chromium GPU 开启并允许 H.264 WebCodecs 快速路径；缺省 `false` 以兼容 CI/xvfb。要启用最快路径，同时写 `"gpu": true` 与 `"hwAccel": "auto"` |
| `videoFrameMode` | | `"fast"`（默认，背景视频正向连续播放追帧，快很多）/ `"exact"`（逐帧精确 seek，慢但同一次导出重跑字节级一致）；只在有背景视频时有意义，见 §11 |
| `style` | | 任意样式字段的子集，未给的用默认值。特效：`effectId`、`effectInDurationMs`、`effectOutDurationMs`（全局 In/Out 毫秒数，同时给出时 In 优先）。文字：`fontFamily`/`fontSize`/`fontWeight`/`italic`/`textColor`/`textAlpha`、`letterSpacing`/`wordSpacing`/`lineSpacing`、`textAlign`（`left`/`center`/`right`）、`textOrientation`（`horizontal`/`vertical`）、`strokeColor`/`strokeWidth`/`strokeAlpha`、`textBgColor`/`textBgAlpha`、`halo`/`glowColor`、`shadowColor`/`shadowAlpha`/`shadowBlur`/`shadowOffset`。背景：`bgType`（"solid"/"gradient"/"image"）、`bgImage`（图片路径，相对 job 目录解析）。全局文字变换：`globalDx`/`globalDy`（像素）、`globalRotate`（度） |
| `lineEffects` | | 键为行序号（`"3"`）或区间（`"0-7"`），值为特效 id（见 §7.2 表） |
| `lineEffectsOut` | | 写法同 `lineEffects`，但设置逐行退场效果；例如 `{ "0-7": "evaporate-out" }` |
| `lineEffectDurations` | | 逐行 In/Out 时长（秒），例如 `{ "0-7": { "in": 0.6, "out": 0.4 } }`。两者同时给出时 In 优先，Out 自动缩短；始终限定在字幕 segment 内且不重叠 |
| `lineStyles` | | 键为行序号或区间，值为文字样式覆盖；字段同 `style` 的文字样式字段，适合 CLI 批量设置某几句字幕的间距、对齐、方向、描边、辉光、底色、阴影 |
| `tracks` | | 额外字幕组（多语言字幕），见下方「多字幕组」小节 |

媒体线段对象字段：

| 字段 | 说明 |
|---|---|
| `path` | 必填，媒体文件路径（相对 job 文件目录） |
| `start` | 时间轴起点（秒），默认 0 |
| `loop` | 重复次数（≥1 的整数，默认 1）或 `"infinite"`（循环到成片结束） |
| `in` / `out` | 源入点/出点（秒）：只取素材的这一段，默认整段 |
| `speed` | 播放速度 0.25–4（音轨变速不变调），默认 1 |
| `layer` | 视频层序：0 最底，高层画面盖在低层上（画中画），默认 0 |
| `x` / `y` / `scale` | 视频画面平移（画布像素）与缩放（铺满画布为 1.0） |
| `fadeIn` / `fadeOut` | 音轨淡入/淡出时长（秒，默认 0 = 无）；从线段起点淡入、到线段结束淡出 |
| `transIn` / `transOut` | 视频进/退场转场（音频忽略）。对象 `{ "type": "fade", "dur": 1 }` 或简写 `"fade:1"`。`type` 取值：`fade`、`slideL`/`slideR`/`slideU`/`slideD`（四向滑动）、`zoom`（缩放+淡）、`wipeL`/`wipeR`（横向擦）。`dur` 为秒 |

**视频间转场**（如 A→B 交叉淡化）= 让 B 的起点重叠到 A 结束之前，并给 B 设 `transIn`：

```json
"video": [
  { "path": "a.mp4", "transOut": "fade:1" },
  { "path": "b.mp4", "start": 9, "transIn": "fade:1" }
]
```

B 在重叠区淡入、盖在 A 之上即得交叉淡化；改 `transIn`/`transOut` 类型可得推入/擦入等。无重叠且两段都设 `fade` 则呈"淡入淡出经黑场"。

把视频文件写进 `audio` 数组即可提取它的音轨（等价 GUI 的「提取音频」）。

独立文字块用顶层 `texts` 字段：

```json
"texts": [
  {
    "text": "标题文字",
    "start": 0.5,
    "end": 5,
    "effect": "fold-in",
    "effectOut": "zoom-away-out",
    "x": 0,
    "y": -500,
    "style": { "textOrientation": "vertical", "strokeWidth": 6, "letterSpacing": 10 }
  }
]
```

`effect` 设置进场效果，`effectOut` 设置退场效果（都使用 §7.2 的特效 id，省略则跟随全局默认）；`x`/`y` 为画面位置偏移（画布像素，相对画面中心）；`style` 可覆盖该文字块的字体、间距、对齐、方向、描边、辉光、底色、阴影等文字样式。

### 多字幕组（双语字幕）

顶层 `lrc` 视为「主字幕组」（位置固定居中，与不写 `tracks` 的旧 job.json 完全一样）；
`tracks` 数组是额外叠加的字幕组，各自一份独立的歌词/时间轴，竖直错开与主字幕组同时显示：

```json
"tracks": [
  {
    "name": "English",
    "lrc": "song.en.lrc",
    "offsetY": 260,
    "lineEffects": { "0-3": "wave-in" },
    "lineEffectsOut": { "0-3": "dissolve-out" },
    "lineStyles": { "0-3": { "fontSize": 60, "textColor": "#ffd400" } }
  }
]
```

| 字段 | 必填 | 说明 |
|---|---|---|
| `name` | | 展示名，仅用于 GUI 里显示，省略则显示占位名 |
| `lrc` | ✔ | 该字幕组自己的歌词/字幕文件路径，写法同顶层 `lrc` |
| `offsetY` | | 相对画面中心的纵向偏移（画布像素）；省略则沿用「新增字幕组」的自动错开位置（依次向下、间距按字号自适应） |
| `visible` | | 省略则可见 |
| `lineEffects` / `lineEffectsOut` / `lineStyles` | | 写法同顶层同名字段，但行序号是**该字幕组自己的**（从 0 数），与顶层行号是两套独立编号——不用管主字幕组占了多少个 id |

`tracks` 数组按顺序依次处理，结果与手动在 GUI 里按同样顺序逐个「+ 新增字幕组」再导入完全一致
（`--save-project` 生成的 `.alicecut.json` 可以验证）。数组顺序很重要：调换顺序会改变各组的默认
`offsetY`（若未显式指定）。

`"audio": "song.mp3"` 这种旧写法仍然有效，等价于 `{ "path": "song.mp3", "start": 0, "loop": 1 }`。
注意：无限循环（`"loop": "infinite"`）的线段不参与时长计算，只是填满到结尾——如果所有素材都无限循环且歌词很短，请显式给 `duration`。

### 输出与退出码

- stdout 逐行：`[export] 37%`（进度）、`[export] done: <路径>`、警告信息
- 退出码 `0` 成功、`1` 失败（错误详情在 stderr）
- 不弹任何窗口，无人值守可用

### 在 Linux CI 中运行

Chromium 需要 display server：`xvfb-run npx electron . --export job.json`。无 GPU 环境没问题（无头模式自动用软件渲染）。注意 pipeline 机器需要装有 job 中指定的字体（或先 `npm run fonts` 用内置字体）。

## 14. 命令控制台

顶栏「命令控制台」在窗口底部展开一条可收起的命令栏：粘贴一段 JSON，Ctrl+Enter（或点「运行」）
立即应用到当前打开的工程，不用离开 GUI、不用另起一个 job.json 文件。

支持 §13 job.json 中适合实时编辑的字段（`lrc`/`tracks`/`audio`/`video`/`texts`/`style`/
`lineEffects`/`lineEffectsOut`/`lineEffectDurations`/`lineStyles`），并额外支持 `select`/`effectIn`/`effectOut`/`effectInDuration`/`effectOutDuration`/`selectedStyle` 选区命令。
不含 `out`/`fps`/`duration`——那三个只在导出视频时才有意义。

```json
{ "style": { "fontSize": 100, "textColor": "#ffd400" } }
```

选中全部字幕后，可在同一条命令中批量设置进场、退场特效和文字样式：

```json
{
  "select": "captions",
  "effectIn": "wave-in",
  "effectOut": "evaporate-out",
  "effectInDuration": 0.6,
  "effectOutDuration": 0.4,
  "selectedStyle": { "fontSize": 100, "textColor": "#ffd400" }
}
```

`select` 还可写 `"all"`、`"none"` 或行 id 数组。时长单位是秒；同一条命令同时给出 In/Out 时 In 优先，Out 被缩短到 segment 剩余区间。`style.riseHistory` 控制上移特效保留的旧字幕数（0–6）。

```json
{
  "tracks": [
    { "name": "English", "lrc": "D:\\song\\song.en.lrc", "offsetY": 260 }
  ]
}
```

一条命令可以同时写多个字段，按固定顺序 `lrc → tracks → audio/video → texts → style →
effectIn/effectOut/effect durations → lineEffects → lineEffectsOut → lineEffectDurations → lineStyles` 依次应用——这样这些行级设置才能命中同一条命令里刚
`lrc`/`tracks` 载入的新行。每个字段各自成功/失败，不是"全对才生效"：某个字段出错只在回显日志
里标红一行，其它字段照常应用；任何一步不满意都可以 Ctrl+Z 撤销，和手动操作没有区别。

**两处与 job.json 命令行模式不同，务必注意：**

- **`lrc` 不会清空工程。** job.json 命令行模式的顶层 `lrc` 是"从零开始渲染一条新片子"，会清空
  当前全部内容；命令控制台的 `lrc` 只是把它非破坏性地替换成新歌词（等价于左侧栏主字幕组的
  「导入歌词到此组…」），不影响已有的其它字幕组、媒体线段、独立文字块——控制台里打错一个字
  不该抹掉正在编辑的工程。真要整个重开，用顶栏「导入歌词」按钮。
- **`tracks` / `audio` / `video` / `texts` 是"追加"，不是"设置"。** 每次运行都会新增一个字幕组/
  线段/文字块，没有"按 id 改已有项"这回事——重复运行同一条命令会重复新增，不是幂等更新。
  只有 `style`/`lineEffects`/`lineEffectsOut`/`lineStyles` 是幂等的。

`lrc`、`tracks[].lrc`、`audio`/`video` 里的文件路径必须是**绝对路径**（`D:\song\a.lrc` 或
`/Users/you/a.lrc`）——控制台没有 job.json 那样的"相对 job 文件目录"概念，相对路径会被当作
文件不存在处理。

## 16. 常见问题

**导入歌词后提示「未找到带时间戳的歌词行」**
文件不是 LRC 格式（每行需有 `[mm:ss.xx]` 前缀），或者是纯文本歌词。

**歌词显示乱码**
GBK 会自动识别；若仍乱码，把文件另存为 UTF-8 再导入。

**字体列表里没有得意黑/霞鹜文楷**
执行 `npm run fonts` 下载（需要能访问 GitHub），或者用「导入字体文件…」手动加载任意 ttf。

**导出失败，提示 FFmpeg 退出码非 0**
看错误日志末尾：常见原因是输出路径不可写、磁盘空间不足。`node scripts/smoke-export.js` 可单独验证编码管线。

**卡点不准**
标准 LRC 只有行级时间，逐字时间是估算的。两个改善手段：① 用带 `<mm:ss.xx>` 逐字标签的增强型 LRC；② 在时间轴上微调每句的起止。

**预览流畅但导出慢**
正常——导出是逐帧离屏渲染 + 编码，不是录屏。60fps 导出耗时约为 30fps 的两倍。

**画面里文字被裁掉**
字号太大或某句太长。排版会自动换行，但停靠转场的侧边/上方堆叠在极端字号下可能出画，调小字号或强度。
