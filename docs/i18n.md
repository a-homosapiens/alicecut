# 多语言（i18n）

界面语言切换。内置 **中文 / English**，并支持**安装语言包**（`.json` 文件）扩展更多语言——经**应用内语言选择器**（顶栏 🌐）切换/安装。

## 架构

i18n 是**纯界面层**：导出的画面渲染的是用户数据（歌词、歌名、字体），永不含 UI 文案，因此切换语言**不影响渲染/导出确定性**。

- **内置字典**：`src/i18n/zh.ts`（source of truth）+ `src/i18n/en.ts`。`en` 类型为 `Record<keyof typeof zh, string>`——少一个键就编译报错（天然的完整性守护）。
- **语言注册表**：`src/i18n/index.ts`——`Locale = string`；`registry` 预置 zh/en，`registerLanguage(id,name,strings)` 注入语言包。`translate(locale,key,params)` 查注册表，**缺键回退中文**（故部分翻译的包也能用），未知语言回退中文；`useT()` 随语言重渲染；`availableLanguages()`/`allKeys()`。
- **当前语言**：Zustand store 的 `locale`（string）+ `languages`（可用语言列表，驱动选择器）。**不写入工程文件**（应用级偏好）。

## 可安装语言包（语言即插件）

语言包是**纯数据 JSON**（无代码 → 无需沙箱），格式：

```json
{ "id": "ja", "name": "日本語", "strings": { "<键>": "<译文>", … } }
```

- **持久化（渲染进程权威）**：`src/i18n/packs.ts` 把已安装包存进 `localStorage`（`alicecut.languages`），上次语言存 `alicecut.locale`；启动时 `registerStoredPacks()` 注册、`loadLocalePref()` 恢复。
- **安装/切换 UI**：`src/components/LanguageMenu.tsx`（顶栏 🌐）——列出内置 + 已安装；「安装语言…」选 `.json` → `parseLanguagePack` 校验 → `installPack`（注册 + 持久化）→ 切换；「导出语言模板…」`makeTemplate()` 生成含全部键、英文初值的起步 JSON 供翻译者/agent 填写。
- **主进程文案**：切换语言时渲染进程经 `app:set-locale` 告知主进程；主进程据此本地化**自己的**原生菜单(File/Edit/View)、文件对话框、窗口标题（仅内置 zh/en，**插件语言回退英文**）。原生菜单已**移除语言子菜单**（切换改在应用内）。
- 示例：`examples/lang-ja.json`（部分日语，演示安装与缺键回退）。

> 局限（v1）：插件语言只本地化**渲染层 UI**（绝大多数），原生菜单/文件对话框/窗口标题对非内置语言回退英文。如需全量，可在包内增加对应键并扩展主进程读取。

## 加一个键

1. 在 `zh.ts` 加 `'区域.键': '中文'`；
2. 在 `en.ts` 加同名键的英文（不加会编译报错）；
3. 组件里 `const t = useT()` 后用 `{t('区域.键')}`，带变量用 `t('区域.键', { n })`。

> 加键后，已安装的语言包对新键自动回退中文，无需更新；翻译者重新「导出模板」即可看到新键。

## 加一种语言

- **语言包（推荐，用户/翻译者）**：顶栏 🌐 →「导出语言模板…」得到含全部键的 JSON，填好 `id`/`name`/译文 → 「安装语言…」导入。纯数据、即时生效、可分发。
- **内置语言（贡献进源码）**：新建 `src/i18n/<lang>.ts`（仿 `en.ts`，键同 `zh`），在 `index.ts` 的 `registry` 预置一项即可。

机制不变——加语言是线性的。复杂复数/数字日期可按需用内置 `Intl.PluralRules`/`Intl.NumberFormat`，无需引库。
