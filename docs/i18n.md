# 多语言（i18n）

界面语言切换。当前支持 **中文 / English**，经由**原生应用菜单**的「语言 / Language」子菜单切换。

## 架构

i18n 是**纯界面层**：导出的画面渲染的是用户数据（歌词、歌名、字体），永不含 UI 文案，因此切换语言**不影响渲染/导出确定性**。

- **字典**：`src/i18n/zh.ts`（source of truth）+ `src/i18n/en.ts`。`en` 类型为 `Record<keyof typeof zh, string>`——少一个键就编译报错（天然的完整性守护）。
- **取译文**：`src/i18n/index.ts` 的 `translate(locale, key, params)`（缺键回退中文→键名；`{name}` 占位插值）与 React 钩子 `useT()`（随语言变化重渲染）。
- **当前语言**：存在 Zustand store（`useProject` 的 `locale` + `setLocale`）。**不写入工程文件**（应用级偏好）。
- **持久化 + 切换 UI**：主进程负责。`electron/menu.ts` 构建原生菜单（语言子菜单为单选），并把语言存到 `userData/settings.json`。
  - 启动：`loadLocale()`（无值则按 `app.getLocale()` 推断）→ `buildMenu` → 页面加载后 `app:locale-changed` 推给渲染进程。
  - 菜单切换：`apply(locale)` 持久化 + 重建菜单（更新选中态与菜单文案）+ 通知渲染进程。
  - 预留 `app:set-locale`（invoke）供未来应用内切换器；菜单本身走 `click→apply`。

## 加一个键

1. 在 `zh.ts` 加 `'区域.键': '中文'`；
2. 在 `en.ts` 加同名键的英文（不加会编译报错）；
3. 组件里 `const t = useT()` 后用 `{t('区域.键')}`，带变量用 `t('区域.键', { n })`。

## 加一种语言

1. 新建 `src/i18n/<lang>.ts`（仿 `en.ts`，键同 `zh`）；
2. `index.ts` 的 `dicts` 与 `LOCALES` 各加一项；
3. `electron/menu.ts` 的 `M` 加该语言的菜单文案，并在语言子菜单加一个单选项。

机制不变——加语言是线性的、只新增文件/条目。复杂复数/数字日期可按需用内置 `Intl.PluralRules`/`Intl.NumberFormat`，无需引库。
